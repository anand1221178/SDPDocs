User Acceptance Tests

  • User Acceptance Tests
    • Incident Reporting Test
      • Given: The user is logged into the app and is on the incident reporting page.
      • When: The user selects an emergency, enters the description and upload a photo.
      • Then: The incident is successfully reported, and a confirmation message is displayed.
    • Notification Test

      • Given: An incident has been reported by another user.
      • When: The user is logged into the app.
      • Then: The user receives a real-time notification with the incident's location and emergency type.
    • Incident List Display Test
      • Given: An incident has been reported by another user.
      • When: The user navigates to the incidents page.
      • Then: The new incident appears on the list with the location and type of emergency.
    • Safety Resources Navigation Test
      • Given: The user is logged into the app.
      • When: The user navigates to the "Safety Resources" section and selects a topic.
      • Then: The app displays the relevant safety information for the selected topic.
    • First Aid Information Navigation Test
      • Given: The user is on the First Aid Information page.
      • When: The user selects available first aid information (e.g.,“CPR”).
      • Then: The app displays steps related to the first aid selected.
    • Real-Time Incident Notification Delay Test
      • Given: Multiple users are logged into the app.
      • When: One user reports an incident.
      • Then: Other users receive a notification in real time without delay.
    • Incident Details View Test
      • Given: An incident has been reported and is listed on the incidents page.
      • When: The user clicks on an incident.
      • Then: The app displays the detailed information of the incident, including the location, type of emergency, and time.
    • Walk-home Assistance Test
      • Given: The user is logged in and on the home page.
      • When: The user navigates to the Walk-home Assistance page, selects "Schedule a ride," enters the time for the ride, and chooses a residence from the list.
      • Then: The user is able to successfully schedule a ride to their chosen residence.
    • Make an Emergency Call Test
      • Given: The user is logged in and on the home page.
      • When: The user presses the “Emergency Call” button, enters the phone number they want to call, and initiates the call.
      • Then: The user is able to successfully make an emergency call.
    • Emergency Information( Access cellphone numbers) Test
      • Given: The user is logged in and on the home page.
      • When: The user navigates to the Emergency Information page, presses the “Wits Contacts” button, and views a list of cellphone numbers.
      • Then: A list of cellphone numbers for Wits Protection Service is displayed.
    • Emergency Information (Make a call) Test
      • Given: The user is logged in and on the Emergency Information page.
      • When: The user presses the “External Contacts” button, views a list of cellphone numbers, and presses "Call" on one of the numbers.
      • Then: The user is able to make a call to the selected external contact.
